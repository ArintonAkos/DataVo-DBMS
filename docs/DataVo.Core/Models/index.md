# Models Overview
The `Models` module acts as the structural vocabulary of the `DataVo.Core` engine, housing all plain C# object (POCO) definitions representing logical SQL constructs. Unlike execution services or the parser, these classes are predominantly stateless data containers representing everything from the physical definitions of tables to the abstract hierarchical nodes representing a fully parsed SQL query.

## Core Responsibilities
* **Catalog Schemas:** Defines the structure of databases, tables, and columns as they logically exist within the metadata definitions.
* **Abstract Syntax Tree (AST):** Encapsulates the hierarchical node expressions generated by the parser.
* **Data Transport:** Serves as the agnostic transport layer bridging parsed tokens to concrete evaluation services without intertwining business logic.

## Component Breakdown

| Component (Directory) | Architectural Role |
|-----------------------|--------------------|
| `Catalog/` | Contains models defining the logical hierarchy of the database server (Databases, Tables, Columns, Indices, and system metadata). |
| `DDL/` | Houses the parsed models representing Data Definition Language commands (e.g., `CREATE TABLE`, `DROP INDEX`). |
| `DML/` | Houses the parsed models representing Data Manipulation Language commands (e.g., `INSERT`, `UPDATE`, `DELETE`). |
| `DQL/` | Houses the parsed models representing Data Query Language commands (primarily the `SELECT` statement and its associated clauses like joins and filters). |
| `Statement/` | Defines overarching programmatic statements, execution blocks, and root command abstractions grouping queries. |

## Dependencies & Interactions
Almost every operational unit in the database depends on the `Models` module. The `Parser` consumes raw characters and deterministically builds trees of `Models`. These models are then handed off to the `Services` and `StorageEngine` layers, which strictly adhere to their structural blueprints to read from tables or evaluate logical filtering conditions. It acts as the universal language across the core architecture.
